# テスト観点（Todo作成）

## 正常系

- titleが1文字以上の場合、Todoが作成される
- titleが最大長の場合、Todoが作成される
- completedは初期状態でfalseになる

## 異常系

- titleが空文字の場合、エラーになる
- titleが最大長以上の場合、エラーになる

## 境界値

- title：1～100文字

# 各テストの位置づけ

## domain テスト

- 単体テスト（クラス仕様テスト）

### 観点：

- 入力 → 内部状態
- 不変条件

## usecase テスト

- 機能テスト（振る舞いテスト）

### 観点：

- ユースケースとして成立するか
- 失敗時に「失敗する」こと
